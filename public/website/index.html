Personal website

Hugo, Markdown, Server Hosting

Motivation

As I am pursuing a career in software development, I recognized the need for a professional online presence—both as a portfolio to showcase my work and as a platform to share my improvements over time.

At the same time, I was curious about self-hosting solutions and wanted to experiment with serving websites directly from my personal server.

This project became an ideal opportunity to achieve both: building a lightweight personal website that doubles as a development blog and a growing archive of my projects and experiments.

Implementation

I chose Hugo as the framework for its simplicity, speed, and excellent support for content written in Markdown—a major time-saver compared to manually writing HTML for each post. The static-site generation approach also aligned well with my self-hosting goals.

The site was deployed on my own server, giving me full control over configuration, routing, and uptime. New content is written in Markdown and processed by Hugo’s templating engine to generate fast, responsive static pages.

Challenges

While Hugo greatly simplified content management, there was a learning curve—especially given that web development isn’t my primary background. The most notable challenge I faced was:

Overuse and over-reliance on HTML

Many custom elements had to be manually written in HTML due to my limited experience with Hugo's templating and partial system.

Over time, I plan to refactor these sections in markdown for better maintainability and consistency.

The Outcome

The site you're viewing now is the result of this project—a living portfolio and developer blog where I share progress, insights, and upcoming projects.

Although the current version includes more hard-coded HTML than I'd prefer, Hugo’s efficiency in creating and managing new content has been a huge productivity boost. Going forward, I'll be incrementally replacing HTML blocks with Markdown and reusable partials to streamline the codebase.
