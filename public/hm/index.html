Hungry Monkey

C, Dynamic programming

Premise

At the beginning of this project I was given code that by using a dynamic array recreted a game wherein a player controlled character had to collect falling elements. The array was populated with 1s and 0s. Every move taken would advance the array by one row

If the character passed through as 1 as it fell you got a point, 0s awarded nothing.

With this foundation I was tasked with creating an algorithm to achieve as high of a score as possible without knowing the array's contents.

Motivation

This idea was originally given as a final project for my C module in college, however my lecturer decided to run a compeition after the submission date to see which student's aglorithm produced the best results. This had me quite interested.

Implementation

I opted for a dynamic programming approach to this problem. As this algorithm would be ran every time a new line from the array appeared on screen it dynamic programming seemed a good choice to use since it focuses around recursive decomposition of problems into smaller problems.



The Alogorithm

Whenever a move needs to be decided my algorithm does 2 main functions.

Firstly it determines what the character can and cannot reach from it's current position.

Secondly for each position the character can reach a calculation is done to find out what the highest possible score from said position is. The highest score for each position is then stored in an array and is refferenced to determine the scores for adjacent positions.

When these calculations finish the cascading calculations will hav formed a triangle of the highest possible value and the character will move to allign itself with the top of this triangle.

Challenges

Inconsistent Tie-breakers

Although my designed algorithm worked very well when there was only a sinlge instance of a given highest number on a given row, it faltered when more than one of the same highest number was present in the same row.

This has shown me the importance of well designed tie-breakers and I'll make sure to improve my methods for such going forwards.

The Outcome

My algorithm provided a satisfactpry high score.

Although it would not consistently produce the highest score, when it differed it was off by an average between 2 and 3 from the highest possible score which still falls within good margins.
